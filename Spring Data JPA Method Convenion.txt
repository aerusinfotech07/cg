# ðŸ”¹ 2. Basic Finder Methods

| Method                          | Meaning                             |
| ------------------------------- | ----------------------------------- |
| `findByName(String name)`       | Select * from employee where name=? |
| `findByEmail(String email)`     | Find employee by email              |
| `findByDepartment(String dept)` | Find employees by department        |

### Example:

```java
List<Employee> findByDepartment(String department);
```

---

# ðŸ”¹ 3. Comparison Keywords

Spring supports many keywords:

| Keyword      | Example                                     | SQL Meaning            |
| ------------ | ------------------------------------------- | ---------------------- |
| GreaterThan  | `findBySalaryGreaterThan(Double salary)`    | salary > ?             |
| LessThan     | `findBySalaryLessThan(Double salary)`       | salary < ?             |
| Between      | `findBySalaryBetween(Double s1, Double s2)` | salary between ? and ? |
| Like         | `findByNameLike(String name)`               | name like ?            |
| StartingWith | `findByNameStartingWith(String prefix)`     | name like 'A%'         |
| EndingWith   | `findByNameEndingWith(String suffix)`       | name like '%A'         |
| Containing   | `findByNameContaining(String word)`         | name like '%word%'     |

---

# ðŸ”¹ 4. AND / OR Conditions

### AND Condition

```java
List<Employee> findByDepartmentAndSalaryGreaterThan(String dept, Double salary);
```

SQL:

```sql
SELECT * FROM employee WHERE department=? AND salary>?
```

---

### OR Condition

```java
List<Employee> findByDepartmentOrSalaryLessThan(String dept, Double salary);
```

# ðŸ”¥ Important Rules

1. Method must start with:

   * `findBy`
   * `readBy`
   * `getBy`
   * `countBy`
   * `deleteBy`
   * `existsBy`

2. Property names must exactly match entity field names.

3. CamelCase is mandatory.

4. Return type can be:

   * `List<T>`
   * `Optional<T>`
   * `Page<T>`
   * `Slice<T>`
   * `Long`
   * `Boolean`
